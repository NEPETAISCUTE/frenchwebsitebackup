L'assembleur est un langage considéré comme de seconde génération, il se base intégralement sur le langage machine, l'idée étant de remplacer les bits par des abréviations (exemple: push pour 0x6A).

L'assembleur utilise une seule étape pour être traduit en langage machine, l'assemblage, convertissant simplement des mots clés en code binaire, le premier assembleur fut inventé par John von Neumann, mais le premier programme assembleur fut écrit par Nathaniel Rochester pour l'IBM 701

Il est très important de savoir qu'il existe un langage assembleur pour chaque famille de processeur, cela va de même pour les langages machines d'ailleurs; par exemple: en x86 10110000 01100001, reviens à mettre la valeur 0x61 dans le registre AL, mais pour une autre famille de processeur, cela serait traduit différemment, ou serait considéré comme étant une instruction illégale.

Il est bon de noter que tout comme il est possible d'assembler du code assembleur, il est possible de désassembler du code machine, neanmoins cela peut être compliqué étant donné qu'il est impossible de différencier données et instructions une fois en binaire, pouvant donc demander une lecture très poussée du code avant de pouvoir le comprendre.

Il faut noter que même aujourd'hui, l'assembleur est utilisé, principalement pour les fondations d'un système d'exploitation, d'un driver (programme de gestion de périphérique), mais, aussi pour le cracking de programme, nécéssitant un désassembleur, ou encore, plus simplement, pour pouvoir bidouiller avec des fonctionalités sinon quasi inaccessibles sur un système (exemple: accéder aux intérruptions BIOS sur un ordinateur avec un processeur x86).
